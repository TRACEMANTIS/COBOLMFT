# JCL anatomy

Job Control Language (JCL) is z/OS's batch description format. Every
line starts with `//`. The three card types you'll meet first:

- **`//JOBNAME JOB`** — accounting, class, message routing.
- **`//STEPNAME EXEC PGM=…`** or `EXEC PROC=…` — invoke a program or a
  procedure.
- **`//DDNAME DD …`** — data definition, attaching a dataset (or
  sysout, or inline data) to a logical name the program reads.

A typical 3-step job looks like:

```jcl
//PAYROLL  JOB  (ACCT),'NIGHTLY',CLASS=A,MSGCLASS=X
//STEP1    EXEC PGM=SORTIN
//SYSIN    DD   *
   SORT FIELDS=(1,9,CH,A)
/*
//STEP2    EXEC PGM=COMPUTE
//INFILE   DD   DSN=PAY.SORTED,DISP=SHR
//OUTFILE  DD   DSN=PAY.RESULT,DISP=(NEW,CATLG)
//STEP3    EXEC PGM=PRINT
//SYSPRINT DD   SYSOUT=*
```

Three steps run sequentially: sort, compute, print. The DD cards bind
external datasets (or in-stream data, or sysout) to the DDNAMEs the
program opens internally.
