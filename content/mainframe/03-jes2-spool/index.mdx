# JES2 spool

JES2 (the Job Entry Subsystem) is the post office of the mainframe. A
submitted job doesn't start running immediately — it joins a queue,
gets converted, gets dispatched, and its output joins another queue.

```
SUBMIT
   │
   ▼
┌─────────┐    ┌──────────────┐    ┌────────────┐    ┌────────┐    ┌───────┐
│  INPUT  │ -> │ CONVERSION   │ -> │  EXECUTION │ -> │ OUTPUT │ -> │ PURGE │
└─────────┘    └──────────────┘    └────────────┘    └────────┘    └───────┘
  raw JCL        translated to       initiator picks    SYSOUT       freed
                 internal text       up, runs steps     spooled
```

- **INPUT** — raw JCL text waiting.
- **CONVERSION** — JES turns it into internal text plus a job log.
- **EXECUTION** — an *initiator* (one of N pre-started address
  spaces) dequeues a job whose CLASS it serves and runs the steps.
- **OUTPUT** — SYSOUT and printer-bound data sit in the output queue.
- **PURGE** — once everything's claimed, the job is purged.

Job classes (set by `CLASS=` on the JOB card) and initiator classes
together control *who* gets to run *what*. A nightly batch job in
CLASS A and an interactive sysprog in CLASS T can share the system
without stepping on each other.
