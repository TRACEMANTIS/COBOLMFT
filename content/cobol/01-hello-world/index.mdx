# Hello, world

Every COBOL program is structured as four **divisions**, in this fixed
order:

1. **IDENTIFICATION** — metadata: program name, author, date.
2. **ENVIRONMENT** — bindings to the outside world: file assignments,
   special-names, the configuration of the host system.
3. **DATA** — every variable you'll touch, declared up front.
4. **PROCEDURE** — the code that actually runs.

For the smallest program possible we only need IDENTIFICATION and
PROCEDURE.

```cobol
IDENTIFICATION DIVISION.
PROGRAM-ID. HELLO.
PROCEDURE DIVISION.
    DISPLAY "HELLO, WORLD".
    STOP RUN.
```

`DISPLAY` writes to standard output. `STOP RUN` ends the program.

<Callout kind="tip" title="Free vs fixed format">
The sample is in *free format*. Classic COBOL is *fixed format* with
columns 1–6 reserved for sequence numbers, 7 for indicators, 8–11 for
area A, 12–72 for area B. Most modern compilers including GnuCOBOL
accept `--free`, which we use throughout this course.
</Callout>
