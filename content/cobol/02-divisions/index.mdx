# The four divisions

A COBOL program is rigidly partitioned. Knowing which division owns
which construct is the fastest way to read unfamiliar code.

| Division        | What lives here                                              |
| --------------- | ------------------------------------------------------------ |
| IDENTIFICATION  | `PROGRAM-ID`, author/date metadata                           |
| ENVIRONMENT     | `FILE-CONTROL`, special-names, configuration                 |
| DATA            | `WORKING-STORAGE`, `LINKAGE`, `FILE SECTION`, copybooks      |
| PROCEDURE       | every executable verb: `PERFORM`, `MOVE`, `IF`, `CALL`, etc. |

Copybooks (the COBOL equivalent of `#include`) are usually pulled into
the DATA DIVISION via `COPY` and rarely into the PROCEDURE DIVISION.
